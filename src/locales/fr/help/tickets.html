Liste des tickets

Titre du ticket
Le numéro du ticket et son titre. Le lien donne accès au détail du ticket.
Date
La date de la dernière modification du ticket.
Projet
Le nom du projet auquel est associé le ticket. Le lien donne accès au détail du projet.
Version
Le nom de la version à laquelle est associé le ticket (la version dans laquelle le ticket sra/a été traité). Le lien donne accès au détail de la version.
Auteur
Le nom de la personne qui a créé le ticket.
Type
Le type du ticket :
  • bug : le ticket se rapporte à un problème rencontré lors de l'utilisation du projet.
  • tâche : le ticket indique une tâhce à effectuer.
  • idée : le ticket propose une idée ou une amélioration pour le projet.
État
L'état du ticket :
  • nouveau : statut par défaut d'un ticket créé.
  • accepté : le ticket a été analysé et accepté.
  • fermé : le ticket à été fermé : indique un ticket résolu, par exemple.
  • supprimé : le ticket a été supprimé, il n'est plus visible dans la partie publique.
  • rejeté : le ticket a été rejeté : il est toujours visible dans la partie publique mais est inactif. Indique un ticket en doublon, ou inapproprié, par exemple.
